Environnement
Logiciel McAfee Email and Web Security (EWS) Appliance 5.x
McAfee Email Gateway (MEG) 7.x

Problème
La recherche de messages sur l'appliance indique que des e-mails ne sont pas remis et les erreurs suivantes s'affichent :
530:Must issue a STARTTLS command first (530 : une commande STARTTLS doit être émise en premier lieu).

Il existe une cause distincte pour chacun de ces codes 5xx :
530:Must issue a STARTTLS command first (530 : une commande STARTTLS doit être émise en premier lieu).
L'appliance reçoit ce message lorsque le serveur de messagerie de destination met en œuvre l'utilisation du protocole TLS (Transport Layer Security) alors que l'appliance ne l'utilise pas.


Solution
Identifiez la raison pour laquelle des e-mails envoyés par le biais du protocole TLS ont été retournés à l'appliance, ainsi que les motifs ou codes d'erreur :

Email Gateway Appliance 7.x 
Connectez-vous à la console de gestion de l'appliance.
Cliquez sur Rapports et Recherche de messages, puis sélectionnez le statut de message Retourné dans la liste déroulante.
Cliquez sur Rechercher/Actualiser.
Vérifiez le code d'erreur de chacun des messages non remis dans la colonne Statut/Catégorie. 
 
Email and Web Security Appliance 5.6
Connectez-vous à la console de gestion de l'appliance.
Cliquez sur Messagerie et Recherche de messages, puis sélectionnez le statut de message Retourné dans la liste déroulante.
Cliquez sur Rechercher/Actualiser.
Vérifiez le code d'erreur de chacun des messages non remis dans la colonne Statut/Catégorie. 
 
 
Utilisez le champ Solution ci-dessous pertinent pour le code 5xx identifié.
Solution
Error 530:Must issue a STARTTLS command first (Erreur 530 : une commande STARTTLS doit être émise en premier lieu). 
Ce message d'erreur est reçu lorsque le serveur de messagerie de destination autorise uniquement les connexions TLS alors que l'appliance n'est pas configurée pour utiliser ce protocole.

Email Gateway Appliance 7.x :
Connectez-vous à la console de gestion de l'appliance.
